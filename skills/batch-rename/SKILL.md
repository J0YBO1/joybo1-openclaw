## When to use
当用户需要批量处理文件名时使用，包括但不限于：
- "批量重命名文件"
- "给文件加前缀/后缀"
- "用正则表达式批量替换文件名"
- "只重命名特定类型的文件"
- "递归处理子文件夹中的文件"
- "预览重命名效果"
- "安全地批量修改文件名"

## Workflow
1. **用户提供参数**：文件夹路径和重命名规则
2. **文件扫描**：根据过滤条件获取文件列表
3. **预览模式**：显示所有文件的「原文件名 → 新文件名」映射
4. **用户确认**：检查预览结果，确认无误
5. **执行重命名**：实际修改文件名，记录操作日志
6. **结果反馈**：显示成功/跳过/失败统计

## 核心功能

### 1. 基本重命名
- **添加前缀/后缀**：在文件名前后添加固定文本
- **完全重命名**：忽略原文件名，使用前缀+序号格式
- **序号控制**：可选择是否添加序号，序号格式为4位数字（如0001）

### 2. 正则表达式替换
- **模式匹配**：使用正则表达式匹配文件名中的特定部分
- **灵活替换**：将匹配的部分替换为指定文本
- **复杂规则**：支持分组、字符类、量词等高级正则特性

### 3. 文件过滤
- **扩展名过滤**：只处理指定扩展名的文件
- **排除扩展名**：排除特定扩展名的文件
- **递归搜索**：处理子目录中的文件

### 4. 安全特性
- **预览模式**：默认只预览不执行
- **冲突检测**：自动跳过已存在的文件名
- **操作日志**：保存重命名记录，便于追溯
- **模拟执行**：干运行模式，测试重命名效果

## 参数说明

### 基本参数
| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `--path` | string | ✅ | - | 要处理的目录路径 |
| `--prefix` | string | ❌ | "" | 要添加的前缀 |
| `--suffix` | string | ❌ | "" | 要添加的后缀 |
| `--full` | flag | ❌ | false | 完全重命名，忽略原文件名 |
| `--no-index` | flag | ❌ | false | 不添加序号（--full模式下无效） |

### 正则表达式参数
| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `--pattern` | string | ❌ | - | 正则表达式模式，用于匹配文件名 |
| `--replacement` | string | ❌ | "" | 替换字符串，与--pattern一起使用 |

### 文件过滤参数
| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `--ext` | string[] | ❌ | - | 只处理指定扩展名的文件（可多个） |
| `--exclude` | string[] | ❌ | - | 排除指定扩展名的文件（可多个） |
| `--recursive` | flag | ❌ | false | 递归处理子目录 |

### 执行控制参数
| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `--preview` | flag | ❌ | true | 预览模式（默认） |
| `--execute` | flag | ❌ | false | 执行重命名操作 |
| `--dry-run` | flag | ❌ | false | 模拟执行，不实际修改文件 |
| `--log` | flag | ❌ | false | 保存重命名日志 |
| `--verbose` | flag | ❌ | false | 显示详细输出 |

## 使用示例

### 示例1：基本前缀添加
```bash
# 预览添加前缀
batch-rename --path "C:\photos" --prefix "vacation_"

# 执行添加前缀
batch-rename --path "C:\photos" --prefix "vacation_" --execute
```

### 示例2：后缀和完全重命名
```bash
# 添加后缀
batch-rename --path "C:\docs" --suffix "_final" --execute

# 完全重命名（自动添加4位序号）
batch-rename --path "C:\images" --prefix "img_" --full --execute
# 效果：file1.jpg → img_0000.jpg, file2.jpg → img_0001.jpg
```

### 示例3：正则表达式替换
```bash
# 替换文件名中的数字
batch-rename --path "C:\files" --pattern "\\d+" --replacement "NUM" --preview
# 效果：photo123.jpg → photoNUM.jpg

# 移除特定字符
batch-rename --path "C:\files" --pattern "[\[\]]" --replacement "" --execute
# 效果：file[1].txt → file1.txt
```

### 示例4：文件过滤
```bash
# 只处理图片文件
batch-rename --path "C:\media" --ext .jpg .png .gif --prefix "img_" --preview

# 排除临时文件
batch-rename --path "C:\project" --exclude .tmp .bak --suffix "_clean" --execute

# 递归处理所有子目录
batch-rename --path "C:\data" --recursive --prefix "archived_" --execute
```

### 示例5：安全操作
```bash
# 预览所有更改
batch-rename --path "C:\important" --prefix "backup_" --verbose

# 模拟执行（不实际修改）
batch-rename --path "C:\important" --prefix "backup_" --dry-run

# 执行并保存日志
batch-rename --path "C:\important" --prefix "backup_" --execute --log
```

## 高级特性

### 1. 序号格式
- 序号固定为4位数字（0000, 0001, 0002, ...）
- 确保文件名排序时的正确顺序
- 在完全重命名模式下强制启用

### 2. 冲突处理
- **文件已存在**：自动跳过，避免覆盖
- **原文件不存在**：标记为失败
- **文件名未变化**：自动跳过

### 3. 操作日志
- 自动生成JSON格式日志文件
- 包含时间戳、参数、操作结果
- 便于审计和撤销操作

### 4. 错误处理
- 目录不存在：清晰错误提示
- 权限不足：友好权限错误
- 正则表达式错误：详细语法错误
- 未知错误：详细堆栈跟踪（--verbose模式）

## 注意事项

### 安全建议
1. **始终先预览**：使用预览模式检查重命名效果
2. **备份重要文件**：在执行前备份关键数据
3. **使用日志功能**：启用--log参数保存操作记录
4. **测试复杂规则**：先用--dry-run测试正则表达式

### 技术限制
1. **文件名长度**：受操作系统文件名长度限制
2. **特殊字符**：某些特殊字符可能不被所有系统支持
3. **编码问题**：确保文件系统支持文件名编码
4. **符号链接**：不处理符号链接，只处理实际文件

### 最佳实践
1. **批量操作前**：先用少量文件测试规则
2. **复杂重命名**：分步骤执行，先预览再执行
3. **正则表达式**：先在正则测试工具中验证模式
4. **文件过滤**：合理使用扩展名过滤，避免误操作

## 故障排除

### 常见问题

#### Q1: 为什么重命名没有执行？
A: 默认是预览模式，需要添加--execute参数才能实际执行。

#### Q2: 如何只处理特定类型的文件？
A: 使用--ext参数指定扩展名，如`--ext .jpg .png`。

#### Q3: 正则表达式不起作用？
A: 使用--preview模式先测试，确保正则表达式正确。

#### Q4: 如何查看详细错误信息？
A: 添加--verbose参数显示详细错误和堆栈跟踪。

#### Q5: 如何撤销重命名操作？
A: 目前不支持自动撤销，但可以通过操作日志手动恢复。

### 错误代码
| 代码 | 含义 | 解决方法 |
|------|------|----------|
| 0 | 成功 | 操作完成 |
| 1 | 目录错误 | 检查路径是否存在，是否有访问权限 |
| 2 | 正则错误 | 检查正则表达式语法 |
| 3 | 其他错误 | 使用--verbose查看详细错误信息 |
